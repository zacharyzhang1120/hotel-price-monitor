## 本地启动

后端 mock 模式：

```bash
cd backend
SCRAPER_MODE=mock python3 -m uvicorn app.main:app --port 8080
```

后端真实抓取模式：

```bash
cd backend
SCRAPER_MODE=real python3 -m uvicorn app.main:app --port 8080
```

扩展：

```bash
cd extension
npm install
npm run dev
```

## 关键注意

- 不要提交 `.env`、数据库、携程登录态、私钥、构建产物。
- `data/ctrip_state.json` 是携程登录态，不在 GitHub 仓库里，需要在实际运行环境单独生成或复制。
- 真实抓取慢的主要瓶颈是携程页面加载和服务器网络，不要用大重构硬改。
- 优先保持现有接口和页面结构，采用小步 Loop：改动、测试、记录、部署。

## 用户当前最关心的问题

1. 切换“我方酒店/门店组”后，要能按当前组抓取 1 家我方酒店 + 5 家竞对酒店。
2. 配置里只输入酒店名称时，希望能自动搜索携程酒店并保存 URL 或 mapping。
3. 抓取结果要确保房型名和价格对应，优先抓取页面上真实的起价房型和起价价格。
4. 日报复制粘贴要可靠。
5. 抓取速度仍慢，需要优先做后台自动抓取、今日优先、失败重试、渐进显示，而不是让用户一直等立即刷新。
