# 酒店竞对价格监控系统 — 产品需求文档（PRD）

> 版本：v1.2 | 日期：2026-07-01 | 状态：MVP 可测试，携程真实抓取已在阿里云跑通 6/6，后台约每 3 小时自动抓今日价，企业微信暂缓

---

## 一、产品概述

### 1.1 产品定位

面向酒店收益管理人员的**轻量级竞对价格监控工具**，帮助用户实时掌握竞对酒店在各 OTA 平台上的起价变化，支持远期价格趋势预判。

### 1.2 核心价值

| 痛点 | 解决方案 |
|------|---------|
| 手动在 3 个平台逐个查价，耗时易错 | 自动化一次抓取，分钟级完成所有对比 |
| 只能看到今天价格，不知道竞对未来调价动态 | 每次抓取今日 + 未来 7 天，形成远期价格日历 |
| 无法感知竞对日内调价 | 约每 3 小时后台自动抓取今日价，降低手动等待成本 |
| 数据散落，无法追溯历史趋势 | 所有价格落库存储，支持历史回溯和趋势分析 |

### 1.3 使用场景

**场景 1：每日价格巡检**
> 收益经理早上到公司，打开浏览器插件，一眼看到昨天最后一次抓取的竞对价格日历。红色单元格表示竞对价格低于我方同平台价格，绿色表示高于我方同平台价格。

**场景 2：调价决策支持**
> 下午 2 点，竞对 A 在携程降了 ¥40。14:00 后台抓取自动对比出来，收益经理据此决定我方是否跟进调价。

**场景 3：远期预判**
> 临近国庆假期，收益经理查看未来 7 天价格日历，发现竞对们从 10 月 1 日开始普遍涨价 30%，据此调整我方远期价格策略。

## 三、功能需求

### 3.1 数据抓取（后端核心）

#### R1 — 携程平台抓取
- 支持平台：携程（Ctrip）
- 抓取目标：每家酒店在携程上的**起价房型名称 + 起价**
- 抓取酒店范围：1 家我方酒店 + 5 家竞对酒店
- 登录方式：需先运行 `python3 scripts/login_ctrip.py` 扫码登录，保存 session 后抓取器复用
- 平台可扩展（设计支持多平台，`config.PLATFORMS` 和 `ENABLED_PLATFORMS` 可随时增删）

#### R2 — 远期价格采集
- 当前测试版默认抓取：**当日入住价格**，用于快速完成今日竞对巡检
- 远期价格支持手动补抓：**未来 7 天入住价格**，不阻塞“立即刷新”和后台今日价任务
- 数据维度：`(酒店, 平台, 入住日期) → (起价房型, 起价)`
- 入住日期与抓取时间分离存储，支持：同一入住日被多次抓取后看到价格日内变化
- 每次抓取生成独立批次（`scrape_run` / `batch_id`），前端和日报默认读取同一批次数据，避免不同平台完成时间不同导致数据混批

#### R3 — 定时调度
- 抓取时间：当前线上为每日 **08:00、11:00、14:00、17:00、20:00、23:00**
- 抓取范围：后台定时默认只抓当前有效门店组的今日价
- 每次抓取完成后自动触发日报生成
- 支持手动触发抓取（浏览器插件中点击按钮）

#### R4 — 反爬策略（轻量版）
- 使用 Playwright 无头浏览器，模拟真实用户行为
- 添加随机延迟（2-5 秒）避免触发频率限制
- 优先分析各平台日历 API 接口，batch 请求减少页面交互次数
- 低频抓取（当前真实抓取只启用携程，6 酒店 × 6 次/天 = 36 个酒店页面/天），代理 IP 非必需

### 3.2 桌面端浏览器插件（首期交付）

#### R5 — 价格日历热力图（核心界面）
```
        6/25  6/26  6/27  6/28  6/29  6/30  7/1   7/2
竞对A-携程  ¥638  ¥658  ¥698  ¥698  ¥728  ¥728  ¥728  ¥728
竞对A-去哪儿 ¥628  ¥648  ¥688  ¥688  ¥718  ¥718  ¥718  ¥718
竞对A-同程  ¥618  ¥638  ¥678  ¥678  ¥698  ¥698  ¥698  ¥698
竞对B-携程  ¥728  ¥728  ¥748  ¥748  ¥768  ¥768  ¥768  ¥768
...
我方-携程   ¥688  ¥688  ¥698  ¥698  ¥718  ¥718  ¥718  ¥718
```
- 单元格颜色深浅映射价格高低（热力图）
- 平台行颜色按同平台比较：竞对某平台价格低于我方同平台价格为红色（威胁），高于我方同平台价格为绿色（安全）
- Y 轴可切换：查看所有酒店 或 单选某家竞对
- 点击某单元格 → 聚焦该天入住价格明细，不再展示历史趋势折线图

#### R6 — 当日价格对比表
- 展示最新一次抓取的当日入住价格
- 表格列：酒店名称 | 平台 | 起价房型 | 起价 | 与我方差价
- 与我方差价默认按同平台价格比较；底部汇总按“竞对最低价 vs 我方最低价”统计威胁数量
- 按价格从低到高排序
- 我方酒店行高亮显示
- 支持一键复制为文本

#### R7 — 运营判断与证据链
- 默认收起，避免巡检页过重
- 展示“谁比我便宜”“是否建议关注调价”“未来价格是否异常”
- 判断依据必须来自当前价格数据和抓取证据，不输出无依据结论
- 每个抓取任务可查看证据：候选房型、候选价格、最终选择、页面信号、耗时和错误原因

#### R8 — 手动刷新
- 顶部显示**上次抓取时间**和**抓取状态**
- 按钮「立即抓取」→ 触发后端 `/api/v1/scrape/trigger`
- 默认只刷新今日价格，远期价格通过单独的“补抓远期”触发
- 抓取过程中允许已完成酒店先展示，抓取完成后自动刷新页面数据

#### R9 — 插件形态
- **独立弹出面板**（点击浏览器扩展图标打开 Popup）
- 不是注入到携程页面的侧边栏（不与 OTA 页面耦合）
- Popup 尺寸建议：800×600px
- 插件内提供「巡检 / 配置」切换：巡检页看价格，配置页维护平台 URL、默认房型、单链接探测、定时状态和手动备份。

---

### 3.3 微信推送（二期交付）

#### R10 — 日报推送
- 每次抓取完成后，推送格式化日报到企业微信
- 日报格式（详见附录 A）
- 推送人群：收益经理 + 店长（企业微信内部群或单人）

#### R11 — 价格异动预警（可选）
- 同一入住日、同一平台的价格，与前一天同一调度时段的最新成功批次对比
- 价格变化超过 ±¥30 或 ±5% 时额外推送预警

## 四、非功能需求

### N1 — 性能
- 单次抓取（6 酒店 × 3 平台 × 8 天）在 60 秒内完成；若降级到 Playwright 页面点击兜底，允许放宽到 3-5 分钟
- API 接口响应时间 < 500ms（数据已在库中）

### N2 — 数据量
- 日写入量：6 × 3 × 8 × 3 = 432 条/天
- 月写入量：~12,960 条，年写入量：~155,520 条
- SQLite 完全满足，无需外接数据库

### N3 — 可靠性
- 单个平台抓取失败不影响其他平台
- 抓取失败不阻塞调度，记录错误日志
- 数据库自动备份（每日一次 copy 到 `data/backups/`）

### N4 — 安全性
- 仅内部使用，不对公网开放
- 后端默认监听 localhost:8080
- 未来若部署到服务器，需加 API Key 认证

### N5 — 可维护性
- 各平台抓取器独立文件，新增平台只需新增一个子类
- 报告格式与数据逻辑分离，换推送通道不改业务代码
- 二期微信推送预估新增代码 < 100 行

## 五、二期复用设计（架构约束）

二期企业微信推送必须**直接复用**一期后端，这是架构设计的核心约束：

| 一期模块 | 二期复用策略 |
|---------|------------|
| 抓取引擎 | 100% 复用，原则上不修改代码 |
| 数据库 | 100% 复用，同一套表结构 |
| 价格 API | 100% 复用 |
| 报告服务 | 100% 复用，已内置 `format=wechat_text` 输出 |
| 定时调度 | 在 `scrape_and_report()` 中增加推送通道调用 |
| 推送通道 | 默认方案：OpenClaw WeCom 插件配置；也可替换为企业微信 Webhook 或自建应用 API |

## 七、技术栈

| 层 | 选型 | 说明 |
|------|------|------|
| 后端框架 | Python 3.9+ / FastAPI | 原生 async，与 Playwright 天然配合 |
| 数据库 | SQLite + SQLAlchemy 2.0 async | 零配置，数据量小无需 MySQL |
| 爬虫 | Playwright (Python) | 无头浏览器，支持反检测 |
| 定时任务 | APScheduler | 轻量 Cron 调度 |
| 扩展框架 | WXT | MV3 支持，Vue/React 集成，HMR 开发体验 |
| 前端框架 | Vue 3 (Composition API) | 学习成本低，ECharts Vue 绑定成熟 |
| 图表 | ECharts 5 | 原生日历热力图 |
| CSS | 原生 CSS | 插件 UI 体量小，当前以明确尺寸和紧凑布局为主 |
| 包管理 | npm (Node 24) + pip/venv | 当前机器已安装 |


**关键设计**：
- `hotels` 表表示真实酒店；平台侧 ID、URL、默认房型名放在 `hotel_platform_mappings`，避免同一家酒店被拆成 3 条业务酒店。
- `check_in_date`（入住日期）与 `scraped_at`（抓取时间）分离。同一入住日期可被不同批次抓取多次，支持观察日内价格变化。
- `batch_id` 绑定同一次抓取产生的所有价格记录。日历、日报、当日对比默认读取同一批次，保证数据口径一致。
- `scrape_task_results` 记录每个批次中每个酒店/平台组合的成功或失败明细，便于真实 OTA 抓取调试和插件端展示。

### 附录 A — 微信推送日报格式

```
📊 6月25日 18:00 竞对价格日报
━━━━━━━━━━━━━━━━━━━━
🏨 我方：亚朵·西湖店
最低起价：同程 尊享大床房 ¥668
平台价格：携程 ¥688 | 去哪儿 ¥678 | 同程 ¥668

竞对价格：
竞对A（0.8km）
  携程 ¥638（较我方同平台 -¥50）
  去哪儿 ¥658（较我方同平台 -¥20）
  同程 ¥628（较我方同平台 -¥40）⚠️最低

竞对B（1.2km）
  携程 ¥728（较我方同平台 +¥40）
  去哪儿 ¥718（较我方同平台 +¥40）
  同程 ¥698（较我方同平台 +¥30）最低

竞对C（0.5km）
  携程 ¥588（较我方同平台 -¥100）
  去哪儿 ¥598（较我方同平台 -¥80）
  同程 ¥578（较我方同平台 -¥90）⚠️最低

⚠️ 低于我方最低价：
竞对A·同程 ¥628（低 ¥40）
竞对C·同程 ¥578（低 ¥90）

📌 近期可关注：竞对C 连续3天低于我方最低价 ¥80+
⏰ 抓取：2026-06-25 18:00 | 下次：22:00
```

**日报价格口径**：
- 我方基准价：取我方在各平台的最低起价，用于“低于我方最低价”和 `vs_mine` 总览判断。
- 平台差价：竞对某平台价格与我方同平台价格比较，用于每个平台价格后的括号说明。
- 竞对最低价：取该竞对在各平台的最低起价，并标记最低平台。
- 企业微信文本使用缩进格式，不使用 Unicode 表格线，避免手机端非等宽字体导致错位。
